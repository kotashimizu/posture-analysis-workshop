# AntiGravityへの作業ルール

このリポジトリは、AntiGravity（GoogleのAI駆動IDE）で開いて使うことを前提としたワークショップ教材です。

受講者が `prompts/01_*.md` から順番に内容をコピーしてAntiGravityに貼ると、姿勢分析Webアプリが完成します。

## あなた（AntiGravity）が守るべきこと

### 基本ルール

1. このフォルダの中だけで作業してください
2. 受講者は **PT・OT・STなどの医療職** であり、プログラミングは初心者です。専門用語は噛み砕いて説明してください
3. プロンプトに書かれた内容を **そのまま** 実装してください。勝手に追加機能や別の方針に変更しないでください
4. 各ステップで「何をするか」を1〜2行で短く説明してから実装に入ってください
5. エラーが出た場合は、原因を初心者向けに説明し、勝手に複雑な回避策へ進まないでください
6. 既存ファイルを編集する場合は、なるべく **既存の構造・命名規則** を踏襲してください

### 技術スタック（固定）

- Vite
- React 19
- TypeScript
- Tailwind CSS v4
- shadcn/ui 風の自前コンポーネント（Radix UIなど外部依存は不要）
- @mediapipe/tasks-vision（CDN経由でモデル読込）
- lucide-react（アイコン）
- HTML5 Canvas（描画）

### 文章ルール

- emダッシュ「—」「──」「ーーー」を一切使わない（コード内コメント・UI文言・ドキュメント全箇所で禁止）
- 区切りや補足には「：」「。」「（）」を使う
- 「診断」「治療」と断定する表現を避け、「傾向」「参考情報」を基本トーンにする

### プライバシー設計

- 画像・動画はすべて **ブラウザ内のみで処理** します
- 外部サーバーへ送信する処理を絶対に書かないでください
- カメラ起動には `getUserMedia` を使い、Stream停止時は必ずトラックを止めてください

### コード品質

- 状態管理は `useState` / `useReducer` のみ
- パスエイリアス `@/` は `src/` にマップ
- 仮閾値・仮値には `// TODO: 要臨床調整` コメントを必ず付ける
- メモリリーク防止：`requestAnimationFrame` のクリーンアップを `useEffect` の return で確実に行う

## ステップごとの進め方

`prompts/01_*.md` から `prompts/09_*.md` まで順番に処理してください。

各プロンプトには以下が書かれています：

- **このステップの目的**
- **AntiGravityが行うこと**（具体的な指示）
- **完成後の確認ポイント**
- **次のステップへ進む条件**

各ステップが完了したら、受講者に「次のプロンプトに進んでください」と促してください。

## 既存ファイル

- `.gitignore`：標準的なNode.js/Vite用
- `prompts/`：受講者が読むプロンプト集（編集禁止）
- `docs/`：背景知識（編集禁止）
- `public/`：受講者が素材を入れる場所
- `_complete-reference/`：完成版アプリ（参照のみ、編集禁止）
- `_instructor/`：講師用メモ（受講者には触れない）

それ以外のファイル（`package.json`、`src/` 配下など）は、Step 01以降で生成していきます。

## 完了条件

`prompts/09_*.md` まで終わった時点で：

1. `npm install` がエラーなく完了している
2. `npm run dev` で開発サーバーが起動する
3. ブラウザで `http://localhost:5173` を開いてアプリが表示される
4. 画像・動画・カメラの各入力モードが動く
5. 姿勢分析の数値レポートが表示される
6. `npx tsc --noEmit` で型エラーが0
7. emダッシュが新規追加されていない

これらが満たされていれば、ワークショップ完了です。
