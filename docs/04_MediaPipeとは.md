# 04 MediaPipeとは

ワークショップで使う姿勢検出技術の概要です。

---

## MediaPipeとは

Googleが2019年から公開しているメディア処理ライブラリ。
画像・動画から、人体・顔・手などのキーポイントを検出します。

### 特徴

- **無料**（Apache License 2.0）
- **商用利用可**
- **ブラウザで動く**（WebAssembly + GPU）
- **オープンソース**

---

## 採用例

皆さんが普段触れているもので、MediaPipe系の技術が使われている例：

- **Snapchat / Instagram / TikTokのARフィルター**：顔・体に重ねるエフェクト
- **YouTubeショートのエフェクト**
- **Google Meetの背景ぼかし**
- **フィットネスアプリ**：フォーム確認機能
- **動作解析の研究**：運動科学・スポーツ分野

---

## このアプリで使うのは「Pose Landmarker」

MediaPipeにはいくつかのモジュールがあります：

| モジュール | 用途 |
|---|---|
| **Pose Landmarker** | 人体の姿勢キーポイント（このアプリで使用） |
| Face Landmarker | 顔の478点 |
| Hand Landmarker | 手の21点 |
| Image Segmenter | 画像セグメンテーション |
| Object Detector | 物体検出 |
| Gesture Recognizer | ジェスチャー認識 |

---

## 33点のランドマーク

Pose Landmarker は1人の人物につき33個のキーポイントを返します。

### 番号と部位

| 番号 | 部位 |
|---|---|
| 0 | 鼻 |
| 1〜6 | 目（左右の内・外端、目頭） |
| 7, 8 | 左耳、右耳 |
| 9, 10 | 口（左、右） |
| 11, 12 | 左肩、右肩 |
| 13, 14 | 左肘、右肘 |
| 15, 16 | 左手首、右手首 |
| 17〜22 | 手指（小指・人差し指・親指） |
| 23, 24 | 左股関節、右股関節 |
| 25, 26 | 左膝、右膝 |
| 27, 28 | 左足首、右足首 |
| 29, 30 | 左踵、右踵 |
| 31, 32 | 左足先、右足先 |

### 各点が持つ情報

```typescript
{
  x: 0.5,            // 画面横位置（0〜1で正規化）
  y: 0.7,            // 画面縦位置（0〜1で正規化）
  z: -0.05,          // 奥行き推定（精度低め）
  visibility: 0.95   // この点がどれくらい確実に見えているか（0〜1）
}
```

---

## visibility の使い方

visibility が低い = 隠れている / 推定が不確か。

このアプリでは：

- **矢状面評価**：visibility が高い側の半身を優先（手前側を採用）
- **視点判別**：左右の visibility 比から「どちら向きか」を推定
- **検出失敗判定**：主要点（鼻・肩・腰・膝・足首）の visibility が0.4未満なら「全身が入っていない」エラー

---

## 強みと限界

### 強み

- 無料で使える
- ブラウザで完結する（サーバー不要）
- 1枚の画像から即座に33点が取れる
- 設置が簡単（CDNから読み込むだけ）
- リアルタイム処理可能

### 限界

- **触診ベースの正確さは無い**：服の上から推定するため
- **ASIS / PSIS / 肩峰 の正確な触診点は取れない**
- **服装で精度が変わる**（タイトな服が好ましい）
- **撮影角度・照明**で精度が変わる
- **奥行き（z値）の精度は低い**：3D解析には不向き
- **服が体を完全に覆うと検出しづらい**

---

## このアプリの位置づけ

「触診の代替ではなく、客観指標を可視化する参考ツール」

具体的には：

- 患者さんへの説明資料として「数値で見せる」
- 経時変化を客観データとして残す
- 経験の浅いセラピストの観察を補助する

最終的な姿勢評価・介入判断は、必ず資格を持つ専門職が行ってください。

## モデルの種類

Pose Landmarker には3つのモデルサイズがあります：

| モデル | サイズ | 速度 | 精度 |
|---|---|---|---|
| **lite** | 約3MB | 速い | 低 |
| **full** | 約9MB | 中 | 中（このアプリで採用） |
| **heavy** | 約26MB | 遅い | 高 |

このアプリは `pose_landmarker_full.task` を使用。
速度と精度のバランスが良いです。

精度を上げたい場合は heavy に変更可能（プロンプトで「heavyに変えて」と頼めばOK）。
