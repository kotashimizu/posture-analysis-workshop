# 02 入力UI構築

## このステップの目的

- 画像/動画を切り替えるタブ
- ファイルをドラッグ&ドロップでアップロードできる入力エリア
- 撮影視点を表示するエリア（中身は次のStepで自動判別を実装）
- メインのレイアウト（入力カード+プレビュー予定エリア+レポート予定エリア）

# やること

## 1. 入力モード切替（画像/動画タブ）

`src/components/ModeTabs.tsx` を作成：
- Tabsコンポーネント（src/components/ui/tabs.tsx）を使う
- value: "image" | "video"
- 「画像」「動画」の2タブ
- onChange でステート更新

## 2. 視点表示コンポーネント

`src/components/ViewSelector.tsx` を作成：
- detectedView: ViewType | null を受け取る
- 上部に「撮影視点（自動判別）」のラベル
- 下部に「自動判別: 矢状面（左向き）」のような表示
- 検出前は「判別前（画像を読み込むと自動判別します）」と表示
- 薄いグレー背景の rounded-xl box

`src/content/view-labels.ts` を作成：
```ts
import type { ViewType } from "@/types";

export const VIEW_LABELS: Record<ViewType, string> = {
  "sagittal-left": "矢状面（左向き）",
  "sagittal-right": "矢状面（右向き）",
  "frontal-front": "前額面（前から）",
  "frontal-back": "前額面（後ろから）",
};
```

## 3. ファイルドロップゾーン

`src/components/FileDropzone.tsx` を作成：
- drag&drop と クリックでファイル選択の両対応
- Props: accept（"image/*" or "video/*"）、onFileAccepted、disabled
- Upload アイコン（lucide-react）+ 「ファイルをドラッグ＆ドロップ、またはクリックして選択」テキスト
- ドラッグオーバー時にハイライト（青いボーダー）
- ドロップ時に onFileAccepted(file) を呼ぶ

## 4. メディア入力統合コンポーネント

`src/components/MediaInput.tsx` を作成：
- Props: mode, onFileAccepted, onImageCaptured, onVideoCaptured, onError
- mode === "image" の時：FileDropzone（accept="image/*"）+ 後でカメラボタンを置く場所を予約
- mode === "video" の時：FileDropzone（accept="video/*"）+ 後でカメラボタンを置く場所を予約
- カメラ機能は次のStep 03で実装するので、いまは「カメラ撮影機能（次のステップで追加）」のプレースホルダ表示でOK
