## 1. カメラ操作のヘルパー

`src/lib/camera.ts` を作成：

```ts
export async function requestCameraStream() {
  if (!navigator.mediaDevices?.getUserMedia) {
    throw new Error("このブラウザではカメラ機能を利用できません。");
  }

  return navigator.mediaDevices.getUserMedia({
    video: {
      width: { ideal: 720 },
      height: { ideal: 1280 },
      aspectRatio: { ideal: 9 / 16 },
      facingMode: "environment",
    },
    audio: false,
  });
}

export function stopCameraStream(stream: MediaStream | null) {
  stream?.getTracks().forEach((track) => track.stop());
}
```

縦長（9:16）を基本にしているのは、PT・OTが患者さんの全身を撮るのに縦撮影が適しているためです。

## 2. CameraCapture コンポーネント

`src/components/CameraCapture.tsx` を作成：

- Props: mode: MediaMode、onImageCaptured: (canvas, previewUrl) => void、onVideoCaptured: (blob) => void、onError: (msg) => void
- ローカルState: active（カメラ起動中か）、recording（録画中か）、localError
- Refs: videoRef、streamRef、recorderRef、chunksRef

機能：

### カメラ起動（startCamera）
- `requestCameraStream()` でストリーム取得
- video要素のsrcObjectに設定して再生
- active = true にする
- エラー時は onError と localError 両方にメッセージ

### カメラ表示エリア
- 縦長プレビュー：`<div className="relative mx-auto aspect-[9/16] w-full max-w-md overflow-hidden rounded-xl bg-slate-950">`
- video要素は h-full w-full object-contain
- mode === "image" の時は「シャッター」ボタン
- mode === "video" の時は「録画開始」「録画停止」ボタン

### 静止画キャプチャ（captureImage）
- video の現在のフレームをcanvasに描画
- canvas.toDataURL("image/png") で previewUrl 生成
- onImageCaptured(canvas, previewUrl) を呼ぶ

### 動画録画（startRecording / stopRecording）
- MediaRecorder でストリームを録画（mimeType: "video/webm"）
- onstop で Blob を生成 → onVideoCaptured(blob)

### クリーンアップ
- useEffect の return で stopCameraStream

### UI
- 「カメラで撮影」セクション見出し
- 説明文：「全身が入るよう縦向き撮影を推奨します。映像はブラウザ内のみで処理します。」
- カメラ起動ボタン（active なら「カメラ再起動」、未起動なら「カメラ起動」）
- localErrorがあれば Alert で表示
- ボタンには Camera / CircleStop / Play / Video アイコン（lucide-react）

## 4. App.tsx の handlers を更新

`src/App.tsx` で：

```tsx
const [source, setSource] = useState<MediaSource | null>(null);
const [video, setVideo] = useState<VideoSource | null>(null);

const handleFileAccepted = (file: File) => {
  if (mode === "image") {
    const url = URL.createObjectURL(file);
    const image = new Image();
    image.onload = () => {
      setVideo(null);
      setSource({ kind: "image", url, element: image });
    };
    image.onerror = () => setAppError("画像を読み込めません。別のファイルを選択してください。");
    image.src = url;
  } else {
    const url = URL.createObjectURL(file);
    setVideo({ url, fileName: file.name });
    setSource(null);
  }
};

const handleImageCaptured = (canvas: HTMLCanvasElement, previewUrl: string) => {
  setVideo(null);
  setSource({ kind: "canvas", url: previewUrl, element: canvas });
};

const handleVideoCaptured = (blob: Blob) => {
  const url = URL.createObjectURL(blob);
  setVideo({ url, fileName: "camera-capture.webm" });
  setSource(null);
};
```

これらのhandlersをMediaInputに渡してください。

プレビュー予定エリア（前回の Card「プレビュー」）には、source があれば画像URLを `<img>` で表示、無ければ「ファイルを読み込むと表示されます」のままにしてください。

## 完了したら確認すること

1. ブラウザリロード

## 詰まったら

- カメラ許可を間違って拒否した場合：ブラウザのアドレスバー左の鍵マークから許可し直す
- HTTPSでないとカメラが使えない場合がある（localhostは例外でOK）
